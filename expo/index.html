Expo App Build Types • Reference Guide

Expo Go Application

Purpose: Fastest way to preview your app. Uses Expo's bundled app with common SDK support (e.g., camera, file system). Ideal for quick testing and sharing.

Characteristics: Live reload support, quick setup, but limited to the bundled SDK version. Does not include React Native debug view. No need for Xcode/Android Studio.

Commands: 
npx expo start
# then scan QR code or press:
#   a → Android emulator
#   i → iOS simulator
#   w → Web browser

Expo Pre‑build (Development Build)

Purpose: Generates native Android/iOS projects based on your app.json. Required to use custom native modules not available in Expo Go.

Characteristics: Full native flexibility, supports live reload and React Native debugging menu. Equivalent to old “eject” process.

Commands: 
npx expo prebuild

# To run:
npx expo run:ios    # iOS simulator/device (Mac only)
npx expo run:android

# Nuke & rebuild native folders:
npx expo prebuild --clean

# Add developer menu:
npx expo install expo-dev-client

Note: macOS is required for iOS development.

Local Native Builds (Xcode/Android Studio or CLI)

Purpose: Use native IDEs or CLI tools to build standalone release apps for submission to App Stores.

Characteristics: Debug development builds with Metro, and produce self-contained release apps. Modify native configs carefully—prefer config plugins.

Process: 
# Run in dev mode from native IDE:
npx expo start  # Metro server

# Release build via CLI:
npx expo run:ios --configuration Release [--device]
npx expo run:android --variant release

# Or use native IDE:
# Xcode → Product → Archive
# Android Studio → Build → Generate Signed Bundle/APK

# Pure Gradle:
cd android && ./gradlew assembleRelease

EAS Builds (Expo Application Services)

Purpose: Cloud-hosted or local build service for producing app binaries easily—including iOS builds on Windows.

Characteristics: Handles provisioning, codesigning, integrates with Fastlane, includes free tier for occasional use. Local builds mimic cloud behavior.

Commands: 
# Log in & configure
eas login
eas build:configure

# Cloud builds:
eas build --profile  --platform 

# Local builds:
eas build --profile  --platform  --local

# Device management (iOS):
eas device:create

Note: iOS local builds still require macOS; cloud builds work cross‑platform.
